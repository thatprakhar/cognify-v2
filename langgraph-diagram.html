Execution Pipeline

03
SlotComposerNode
gpt-4o × N slots LLM · sequential loop
▾
Inputs
state.intentQueryIntentSubject, domain, constraints for prompt context
state.layoutPlanLayoutPlanWhich slots to generate and which module goes in each
state.querystringOriginal user query (included in each slot prompt)
Outputs
slotDraftsRecord<slot, any>One raw config object per slot, keyed by slot name
Internal Logic
1
SystemMap always goes first — its node IDs are captured and injected into downstream RiskPanel/ModuleCards prompts
2
For each slot: looks up SLOT_PROMPTS[moduleName] — a detailed schema-shaped JSON template with all required fields
3
Calls GPT-4o with: "Return ONLY raw JSON, no fences". Strips fences, JSON.parses the response
4
On parse failure, the slot is silently skipped (no error thrown) — validator will catch it

SlotValidatorNode
Deterministic · Zod
▾
Inputs

state.layoutPlanLayoutPlanModule name for each slot (needed to pick the right Zod schema)
Outputs

Internal Logic
1

Filters out "Spec must have…" count errors (single-module temp spec always fails those). Only real schema errors are kept

05
SlotRepairNode
gpt-4o conditional
▾
Inputs
state.slotErrorsRecord<slot, string[]>Zod errors to fix per slot
state.slotDraftsRecord<slot, any>Current invalid configs (shown to LLM for context)
state.slotAttemptsRecord<slot, number>Guards against infinite loops — skip slots at ≥ 2 attempts
Outputs
slotDraftsRecord<slot, any>Merged with updated (repaired) configs
slotAttemptsRecord<slot, number>Incremented per repaired slot
Internal Logic
1
For each slot with errors where attempts < 2: builds a targeted repair prompt with errors + current invalid JSON
2
Calls GPT-4o: "Fix ONLY the issues listed. Return corrected JSON."
3
Parses response. Updates slotDrafts[slot] and increments slotAttempts[slot]
↑ loops back to SlotValidatorNode
exits when all errors resolved OR attempts ≥ 2
